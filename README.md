# ♟️ Aura Chess — Сад Стратегий

**Aura Chess** — это инновационная платформа для геймифицированной когнитивной тренировки, скрытая под видом эстетичной браузерной шахматной игры. Каждая партия здесь — это «сессия размышлений», которая развивает ваш личный виртуальный Сад (RPG-прогрессия). 

Проект использует связку современных веб-технологий и Искусственного Интеллекта, чтобы анализировать ваши ходы с помощью **Stockfish 16** (на базе WebAssembly) и превращать ошибки в понятные обучающие метафоры.

## ✨ Ключевые возможности
- **Живая Шахматная Арена**: Плавное управление доской (на базе `chess.js` / `react-chessboard`) и 100% валидация ходов. Доступны матчи против друга на одном устройстве или против ИИ.
- **Глубокий Анализ**: Stockfish оценивает слабости и ошибки, а Gemini AI оборачивает это в «смысловую» обратную связь.
- **Когнитивная RPG-система (Сад)**: Ваш уровень — это ваше древо. Накапливайте XP за матчи и наблюдайте эволюцию от семечка до мифического древа мудрости.
- **Интегрированная локальная система профилей**: Надежная авторизация и сохранение статистики (победы, монеты, XP) прямо в `localStorage` без привязки к внешним базам данным.
- **Смена тем (Dark/Light mode)** и эстетичный, современный UI (TailwindCSS, Framer Motion).
- **Pro Подписка**: Заготовка дизайна под монетизацию с доступом к эксклюзивным темам доски и безлимитному метафоричному анализу.

## 🛠 Вызовы и Решенные Проблемы в ходе разработки

Во время создания и адаптации этого приложения для Production-окружения (Vercel) мы столкнулись с рядом интересных технических барьеров. Ниже собран лог главных проблем и способов их решения:

### 1. 🐛 Ошибка "Бесконечная загрузка ИИ" (AI не делает ход)
- **Симптом**: При попытке начать игру против ИИ зависала надпись "Загрузка движка...", либо во время партии ИИ ничего не отвечал.
- **Почему это происходило**:
  1. В `next.config.ts` по умолчанию работали строгие заголовки CSP (Content Security Policy). Они блокировали инициализацию WebAssembly-модулей движка (`wasm-unsafe-eval`) и Web Workers (`worker-src`). Движок просто "убивался" браузером.
  2. Архитектура fallback-логики ожидала ответ от сломанного воркера (целых 5 секунд на каждый ход). Ошибка инициализации не обнуляла ссылку на экземпляр воркера.
- **Как решили**:
  - Обновили CSP в `next.config.ts`, добавив `'wasm-unsafe-eval'` для скриптов и `worker-src 'self' blob:`, разрешив браузеру запускать Stockfish.
  - Добавили жесткий таймаут и обработчик `worker.onerror` в кастомный хук `useStockfish`. Теперь, если WebWorker блокируется или падает, ссылка на него уничтожается, и игра *моментально* (без 5-секундных фризов) переключается на запасную логику (случайный ход ИИ), сохраняя играбельность.

### 2. 🐛 Ошибка "Сбой сборки на Vercel (Type Error)"
- **Симптом**: Код отлично генерировался и работал локально через `npm run dev`, но облачный деплой на Vercel постоянно падал на этапе `next build`.
- **Почему это происходило**: В ходе перевода проекта с облачного Supabase Auth на локальную авторизацию (Local Storage Auth Modal), из интерфейса `UserContextValue` были удалены старые методы (например, `loginWithGoogle`). Однако на странице Dashboard (`/dashboard/page.tsx`) старый импорт разрушался, вызывая строгую ошибку проверки типов (TS Type check failed), запрещая компиляцию продакшн сборки.
- **Как решили**: Провели ревизию использования контекста, спустились в проблемные файлы и удалили неиспользуемые переменные-заглушки (unused variables) при деструктуризации хука `useUser()`.

### 3. 🐛 Проблема с Supabase Auth / Авторизацией
- **Симптом**: Ошибка `ERR_EMPTY_RESPONSE` при попытке тестировать регистрацию через GitHub или сторонние провайдеры на начальных этапах. Изначально мы планировали сделать онлайн-базу.
- **Почему это происходило**: Supabase требовал строго настроенных Callback-адресов в облаке, которые расходились с localhost процессом, вызывая десинхронизацию сессий.
- **Как решили**: Временно (или как основную "offline" фичу) написали чистую, отвязанную от сети систему авторизации (Modal Component + Context). Теперь проект работает на 100% автономно на клиенте через хранилище браузера (Local Auth & Stats), что кардинально упростило демо проекта и локальную игру.

### 4. 🐛 Пропадающие стили и Security Warnings на Vercel
- **Симптом**: При первых развертываниях выпадали предупреждения о нехватке секьюрити и возможных XSS.
- **Как решили**: Настроили надежный `next.config.ts`, внедрив `Strict-Transport-Security`, `X-Content-Type-Options` и `X-XSS-Protection`, что улучшило оценку безопасности до Production-стандартов без ущерба для стилей из-за правильной настройки `style-src 'unsafe-inline'`.

## 🚀 Как запустить проект

1. `npm install` — Установка зависимостей.
2. `npm run dev` — Запуск отладочного сервера.
3. Откройте [localhost:3000](http://localhost:3000) и наслаждайтесь игрой!

*(Файл Stockfish.js автоматически копируется в public/ из node_modules скриптом `postinstall`)*
